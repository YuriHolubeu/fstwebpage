## Quasar CLI â€” do I need to install it?

**No global install is required** for normal work on this repo.

After dependencies are installed, the Quasar CLI from **`@quasar/app-vite`** (devDependency) is available as:

- **`npm run dev`** â€” development server  
- **`npm run build`** â€” production build  

Those scripts run the local `quasar` binary from `node_modules/.bin`.

**Optional:** install the global CLI if you want the `quasar` command everywhere in your shell:

```bash

```

Then you can run `quasar dev` / `quasar build` from the project root instead of `npm run â€¦`. Functionally it is the same as using the npm scripts after `npm install`.

## Supabase contact form

Create `.env.local` from `.env.example` and fill in the public project URL and anon key from Supabase:

```bash
cp .env.example .env.local
```

The contact dialog inserts rows into `contact_messages` by default. Create this table in Supabase:

```sql
create table public.contact_messages (
  id bigint generated by default as identity primary key,
  created_at timestamptz not null default now(),
  name text,
  email text not null,
  message text not null,
  source text
);

alter table public.contact_messages enable row level security;

create policy "Allow public contact submissions"
on public.contact_messages
for insert
to anon
with check (
  length(email) > 3
  and length(message) > 0
);
```
